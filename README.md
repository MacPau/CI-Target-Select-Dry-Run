# Comet Interceptor Target Selection Dry Run: SPICE Kernels

This repository contains the trajectory kernels for the three anonymized comet
targets used in the Comet Interceptor target-selection dry-run exercise.

The targets are based on historical comet orbits shifted into the time period
relevant to the exercise. The labels `Comet 1`, `Comet 2`, and `Comet 3` are
exercise identifiers; they are intentionally not linked here to the historical
objects on which the simulated trajectories were based.

## Repository contents

| Exercise target | SPK kernel | NAIF target ID | Time coverage (TDB) |
| --- | --- | ---: | --- |
| Comet 1 | `COMET 1/-1000001.bsp` | `-1000001` | 2026-01-01 to 2036-12-31 |
| Comet 2 | `COMET 2/-1000002.bsp` | `-1000002` | 2027-01-01 to 2037-12-31 |
| Comet 3 | `COMET 3/-1000003.bsp` | `-1000003` | 2026-01-01 to 2036-12-31 |

The repository also includes `naif0012.tls`, the NAIF leap-seconds kernel used
to convert UTC calendar dates to SPICE ephemeris time.

Each binary SPK (`.bsp`) contains one Type 9 trajectory segment giving the
comet's geometric state relative to the Sun in the `J2000` reference frame.
SPICE returns position in kilometres and velocity in kilometres per second.
Queries must fall within the coverage interval of the selected kernel.

Because these SPKs are Sun-centred, no planetary ephemeris is needed for the
heliocentric queries shown below.

## MATLAB example

This example uses
[MICE](https://naif.jpl.nasa.gov/naif/toolkit_MATLAB.html), the MATLAB interface
to NASA's SPICE Toolkit.

Run the example from the repository root, or replace the relative kernel paths
with absolute paths.

```matlab
% Select a target and a UTC date within its kernel coverage.
comet_kernel = fullfile('COMET 1', '-1000001.bsp');
comet_id = '-1000001';
utc_date = '2030-10-01T00:00:00';

% Load the included leap-seconds kernel and comet SPK.
cspice_furnsh('naif0012.tls');
cspice_furnsh(comet_kernel);

% Get the geometric heliocentric state in the J2000 frame.
et = cspice_str2et(utc_date);
[state, light_time] = cspice_spkezr(comet_id, et, 'J2000', 'NONE', 'SUN');

position_km = state(1:3);
velocity_km_s = state(4:6);

cspice_kclear;
```

Change `comet_kernel` and `comet_id` together to query another target:

```matlab
% Comet 2
comet_kernel = fullfile('COMET 2', '-1000002.bsp');
comet_id = '-1000002';

% Comet 3
comet_kernel = fullfile('COMET 3', '-1000003.bsp');
comet_id = '-1000003';
```

## Python example

Install the Python SPICE interface with `pip install spiceypy`.

Run the example from the repository root, or replace the relative kernel paths
with absolute paths.

```python
import spiceypy as spice

# Select a target and a UTC date within its kernel coverage.
comet_kernel = "COMET 1/-1000001.bsp"
comet_id = "-1000001"
utc_date = "2030-10-01T00:00:00"

# Load the included leap-seconds kernel and comet SPK.
spice.furnsh("naif0012.tls")
spice.furnsh(comet_kernel)

# Get the geometric heliocentric state in the J2000 frame.
et = spice.str2et(utc_date)
state, light_time = spice.spkezr(
    comet_id, et, "J2000", "NONE", "SUN"
)

position_km = state[:3]
velocity_km_s = state[3:]

spice.kclear()
```

For Comet 2 or Comet 3, change both variables as follows:

```python
# Comet 2
comet_kernel = "COMET 2/-1000002.bsp"
comet_id = "-1000002"

# Comet 3
comet_kernel = "COMET 3/-1000003.bsp"
comet_id = "-1000003"
```

The examples use aberration correction `NONE`, so the returned state is the
geometric state of the comet relative to the Sun at the requested epoch.
